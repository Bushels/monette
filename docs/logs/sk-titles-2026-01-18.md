## Purpose

These CSVs are an ISC-style title export listing land titled to MONETTE FARMS LTD. They give us a **single court-grade authoritative anchor** for what was titled to MFL during CCAA proceedings. The map's parcel layer is currently driven by per-property XLSX files on `G:\` (per `parcel_pipeline.md`); this update reconciles the SK portion of that pipeline against the CSV anchor truth.

This is **not** the complete Monette footprint. Other Monette entities, leased land, and rented-back blocks (Hafford precedent) are not in title records.

## Methodology

### Anchor-truth principle
- **CSV says titled to MFL** → highest confidence; goes on map as MFL-owned (verified)
- **Current data has parcel, CSV doesn't** → keep on map but flag for follow-up; do NOT silently delete
- **CSV has parcel, current data doesn't** → add to map under correct property
- **CSV says titled, but court doc says sold by 2026-01-18** → court doc wins; flag for follow-up

### Per-area workflow (each area processed identically)
1. Extract CSV rows for the area's RM(s)
2. Pull current quarters-data.js entries for the property
3. Generate diff (ADD / KEEP / REMOVE / FLAG)
4. Browser smoke test that parcels render in expected location
5. Per-area Codex review
6. Apply knowledge layer (status, rumors, court figures)
7. Log entry committed before moving to next area

### Codex usage discipline
- Codex 5.5 / xhigh for hard architect work
- Codex gpt-5.5 / medium for routine reviews
- Each Codex round documents its findings into per-area log
- Per memory: 2–3 Codex rounds total budget, plus per-area spot-checks on flagged items only

## Codex round-1 worked example — findings

### Scope

Worked example completed for `wymark` + `swift-current` only. No edits were made to `quarters-data.js` or `data.js`; the reviewable delta package is `docs/logs/sk-titles/wymark/proposed-deltas.json`.

### Architectural surprises for Waves A-E

- Audit counts are title-row counts, but the runtime map mostly renders legal-location features. Wymark has 131 XLSX title rows, 101 unique legal locations, and 101 current map features.
- Wymark's 82 audit FLAGs are not all true follow-up flags. Swift Current's audit proves 17 title rows inside that set are reassignments out of Wymark, collapsing to 16 unique current map features.
- `NW-7-13-14-W3` is the key duplicate-title example: two Swift Current CSV parcel numbers point to one current Wymark feature.
- LSD rows cannot safely reuse quarter geometry. Several Swift Current CSV LSDs matched current Wymark quarter or partial-quarter records, so final application needs DLS geometry computation.
- Point-only court-file properties can receive CSV-confirmed geometry without reconciling their whole court-file acreage claim. Swift Current's 31 CSV rows do not explain the full 49,775 acres in `data.js:808`.

### Risks and unverified assumptions

- The RM-only crosswalk is adequate for this pair only because RM Swift Current 137 CSV rows are T13/R13-R14/W3. Future waves need township/range guards before property assignment.
- Wymark Section 18-12-14-W3 feedlot parcels are absent from the MFL CSV despite older local/title-package context. Treat public ownership copy as unresolved until fresh ISC title pulls close it.
- Carefoot/Waldeck remains a knowledge-layer flag, not CSV-confirmed MFL title. The five named quarters total 746.70 titled acres, while the court-affidavit Phase 2 line is 485 acres.
- Duplicate legal locations in ADD buckets may create overlapping polygons if Claude applies one map feature per CSV title row without aggregation.

### Recommended wave/order changes

- Process reassign-linked pairs before finalizing either property alone. For this wave, Swift Current must be resolved before Wymark's final flag count is trusted.
- Put point-only properties with large ADD/LSD buckets early in the wave, because they expose geometry-generation assumptions before bulk application.
- Add a post-audit normalization step that reports both `csv_title_rows` and `unique_map_features` for ADD/KEEP/REASSIGN/FLAG.
- Add a `reassign_out` view to `scripts/audit_isc_vs_quarters.py`; the current markdown only surfaces REASSIGN from the target property's perspective.

### Crosswalk recommendations

- Keep `RM OF SWIFT CURRENT NO. 137 -> swift-current` for T13/R13-R14/W3 CSV rows from this snapshot.
- Do not generalize that RM rule to all current Wymark legacy records. Wymark still has 28 residual RM Swift Current 137 features absent from the Swift Current CSV set.
- Extend `scripts/parse_isc_titles.py` so low/medium-confidence RMs can use township/range predicates, not just RM name.
- Preserve `RM OF LAC PELLETIER NO. 107 -> wymark` for the confirmed T11/R13-R14/W3 Wymark CSV core.
