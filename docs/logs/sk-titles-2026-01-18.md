## Purpose

These CSVs are an ISC-style title export listing land titled to MONETTE FARMS LTD. They give us a **single court-grade authoritative anchor** for what was titled to MFL during CCAA proceedings. The map's parcel layer is currently driven by per-property XLSX files on `G:\` (per `parcel_pipeline.md`); this update reconciles the SK portion of that pipeline against the CSV anchor truth.

This is **not** the complete Monette footprint. Other Monette entities, leased land, and rented-back blocks (Hafford precedent) are not in title records.

## Methodology

### Anchor-truth principle
- **CSV says titled to MFL** → highest confidence; goes on map as MFL-owned (verified)
- **Current data has parcel, CSV doesn't** → keep on map but flag for follow-up; do NOT silently delete
- **CSV has parcel, current data doesn't** → add to map under correct property
- **CSV says titled, but court doc says sold by 2026-01-18** → court doc wins; flag for follow-up

### Per-area workflow (each area processed identically)
1. Extract CSV rows for the area's RM(s)
2. Pull current quarters-data.js entries for the property
3. Generate diff (ADD / KEEP / REMOVE / FLAG)
4. Browser smoke test that parcels render in expected location
5. Per-area Codex review
6. Apply knowledge layer (status, rumors, court figures)
7. Log entry committed before moving to next area

### Codex usage discipline
- Codex 5.5 / xhigh for hard architect work
- Codex gpt-5.5 / medium for routine reviews
- Each Codex round documents its findings into per-area log
- Per memory: 2–3 Codex rounds total budget, plus per-area spot-checks on flagged items only
